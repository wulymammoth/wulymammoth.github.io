Elixir Pipe Into Assertion

Elixirtesting

139 Words

2020-07-05 16:40 -0700

Love the pipe operator? Miss them in tests with ExUnit when the result is derived from a pipeline of operations necessitating an intermediate variable, like result? I did…

typical test assertion (no pipe operator)

defmodule AdderTest do
  test "add" do
    result =
      1..3
      |> Enum.map(& &1 * 2) # [2, 4, 6]
      |> Adder.add(1) # [3, 5, 7]

    assert result == [3, 5, 7]
  end
end

verbose (with pipe)

Uses an anonymous function - NOTE: must invoke with dot-syntax

defmodule AdderTest do
  test "add" do
    1..3
    |> Enum.map(& &1 * 2)
    |> Adder.add(1)
    |> (fn result -> result == [3, 5, 7] end).()
  end
end

terse (with pipe)

I favor this because it’s terse and concise

defmodule AdderTest do
  test "add" do
    1..3
    |> Enum.map(& &1 * 2)
    |> Adder.add(1)
    |> (& &1 == [3, 5, 7]).()
  end
end
